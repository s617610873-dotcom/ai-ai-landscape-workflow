二、配方库（5 个模板文件）

文件1：01_提取参考方案框架.txt

用途：从参考方案 PDF 提取完整文档结构和文本风格。
使用场景：对话1，上传 PDF 后直接发。

请分析我上传的这套完整的景观设计方案PDF（含所有页面），
按以下权重顺序提取参考框架，输出纯文本，不要markdown标记。

第一部分：文档结构（权重最高，必须完整）
- 总共有多少页
- 逐页列出：页码 + 页面类型
  （封面/目录/设计概念/总平面图/分析图/节点效果图/剖面图/植物配置/小品设施/区位分析/人群分析/专项设计/封底/其他）
- 每页的内容组合方式（纯图/图+文字/图+表格/多图拼版……）
- 页面之间的逻辑衔接关系

第二部分：文本框架与叙事风格（权重高）
- 设计说明的撰写结构（总-分-总/概念先行/问题导向）
- 段落长度和句式特征（长段落描述/短句要点/编号条款）
- 专业术语的使用风格（偏学术/偏通俗/偏感性）
- 每个章节的标题风格（动词开头/名词短语/问句）
- 典型段落的开头句式

第三部分：页面排版特征（权重中）
- 图文比例（每页图占多少、文字占多少）
- 文字块的位置规律
- 图片的呈现方式（满版出血/带白边/带标注/带图例）

第四部分：视觉风格特征（权重低，简要提取）
- 色彩主调
- 渲染风格
- 主要字体
- 代表性设计语汇

注意：
- 图片页标注[图片页：描述画面内容]
- 优先保证文档结构的完整性
- 输出纯文本，不要markdown标记

文件2：02_提取项目数据.txt

用途：从项目 PDF 提取结构化数据。
使用场景：对话1，接着上一条消息发。

请从以下项目PDF的文字内容中提取景观设计所需的关键信息，
按下面10个字段输出纯文本（每行一个字段，不要任何格式标记、不要列表符号）：

1. 项目名称：
2. 红线面积（ha）：
3. 建筑布局简述（建筑数量、层高、主要出入口）：
4. 场地竖向高差（多少米、坡向）：
5. 地下车库出入口位置：
6. 硬景/软景比例（若有）：
7. 主要使用人群：
8. 周边环境（道路、河道、相邻建筑等）：
9. 设计限制条件（管线、退线、古树、保留建筑等）：
10. 甲方特殊要求（风格偏好、造价限额、工期等）：

【项目PDF文字内容】
（粘贴项目PDF导出的纯文本）

文件3：03_小流程出图指令.txt

用途：生成小流程5类核心图纸的 AB 双版提示词。
使用场景：对话2，粘贴项目数据后发。

请基于以下【项目数据】和【参考框架（可选）】，生成一套景观设计核心图纸
的 GPT Image 2 提示词（英文）。

每张图出两个版本：
- Version A：带标注/图例/文字说明的交付版
- Version B：纯图版，无文字，用作WPS素材

以下 S 标签用于后续大流程引用，请在每个提示词前标注：
[S01] [S02] [S03] [S04] [S05]

图纸清单：

图1 [S01]：现状平面图
- 基于项目实地条件生成航拍或CAD底图风格的现状平面
- 标注红线范围、周边道路、相邻建筑
- Version A：red line boundary, labeled surrounding roads and buildings, site area noted
- Version B：same composition, remove all labels and text

图2 [S02]：彩平图（大色块功能分区）
- 大色块清晰区分绿化/硬化/交通/活动区/水景
- 纯色填充，边缘清晰，不要渐变
- 有初步路网骨架
- Version A：color legend in corner, zone labels in Chinese
- Version B：same color blocks, remove legend and labels

图3 [S03]：总平面图（细化方案）
- 在彩平基础上细化植物组团、铺装示意、小品位置、水景形态
- 增加细节层次和材质示意
- Version A：complete with legend, scale bar, north arrow, labels in Chinese
- Version B：same detailed plan, remove all annotations

图4 [S04]：分析图（交通分析/景观结构分析/竖向分析）
- 根据项目特征选择最需要的分析类型
- 用箭头、色带、符号表达分析逻辑
- Version A：clear legend, arrow annotations, analysis labels in Chinese
- Version B：same analysis graphic, remove text

图5 [S05]：效果图/鸟瞰图（核心节点）
- 选取方案中最具代表性的节点
- 人视点或鸟瞰，根据节点特征决定
- Version A：project name overlay, design concept label
- Version B：same rendering, no text overlay

技术要求：
- 5张图的色调和渲染风格保持一致
- 分辨率适配16:9 PPT页面

【项目数据】
（粘贴文件02输出的项目数据）

【参考框架（可选）】
（如果有，粘贴文件01输出的框架描述）

文件4：04_生成大流程分页内容.txt（含显式映射）

用途：生成完整汇报PPT的每页内容，并与小流程图纸做映射。
使用场景：对话3，粘贴框架+数据+映射规则后发。

=== 第一步：图纸映射（必做） ===

以下是小流程的5类图纸标签，每个标签代表一张成品图，
B版是纯图素材，A版是带文字的设计确认稿：

S01 = 现状平面图
S02 = 彩平图（大色块功能分区）
S03 = 总平面图（细化方案）
S04 = 分析图（交通/结构/竖向）
S05 = 效果图/鸟瞰图

请根据【参考方案框架】的页面结构，逐页判断并输出映射结果：

PAGE 1：[页面类型] → [复用 S0X_B.png，参考 S0X_A.png / 需新出图]
PAGE 2：...
（以此类推，覆盖框架中的每一页）

=== 第二步：内容生成（基于映射结果） ===

对于标注"复用"的页面：
- 只需输出该页的文字内容（标题+正文+图注）
- 不要生成图片描述

对于标注"需新出图"的页面：
- 输出完整文字内容
- 同时输出图片描述（画面内容、构图建议）

整体要求：
- 严格遵循【参考方案框架】的文档结构
- 叙事风格模仿参考方案
- 内容用【项目数据】替换
- 每页用 ---PAGE BREAK--- 分隔
- 输出纯文本

【参考方案框架】
（粘贴文件01输出的框架描述）

【项目数据】
（粘贴文件02输出的项目数据）

文件5：05_大流程补图提示词.txt

用途：仅对标注"需新出图"的页面生成 AB 双版提示词。
使用场景：对话3，紧接着上一条消息发（同一个对话）。

以下是【大流程分页内容】中所有标注"需新出图"的页面清单。
请为每一页生成一对 GPT Image 2 提示词（英文），每对包含：
- Version A：文字直接烤在图中的完整方案页

S04_分析图
S05_效果图/鸟瞰图

需要新出图的页面清单：

三、三组对话的规划

对话1：PDF 分析

操作开新对话？为什么
上传参考方案PDF + 发01模板新对话 1上传文件需要干净上下文，避免被之前的对话干扰
在同一对话里，上传项目PDF + 发02模板不另开同一个分析会话，上下文延续
下载保存：框架描述.txt + 项目数据.txt——

对话2：小流程出图（核心设计图）

操作开新对话？为什么
粘贴项目数据 + 发03模板新对话 2从文本分析切换到图像生成，上下文的思维模式完全不同
AI 返回5对提示词（S01-S05）不另开同一个会话
逐张出图：先出 A 版确认，满意后再出 B 版不另开逐张在会话里完成
下载保存：S01_A ~ S05_A + S01_B ~ S05_B——
如果 AI 返回的提示词不满意，直接在同一个对话里让 AI 修改不另开修改依赖上下文

对话3：大流程内容 + 补图

操作开新对话？为什么
粘贴框架 + 项目数据 + 发04模板新对话 3切换到方案内容规划，和小流程对话隔离
AI 完成映射 + 输出完整分页内容不另开—
下载保存：方案分页文本.txt——
继续同一个对话，粘贴方案分页文本 + 发05模板不另开05 依赖 04 的映射结果，开新对话 AI 就不知道映射关系了
AI 返回补图提示词不另开—
逐张出补图（P01-P0N）不另开—
下载保存：P01_A ~ P0N_A + P01_B ~ P0N_B——

WPS 合稿（离线）

操作
打开 WPS，新建 PPT
先铺小流程 B 版图：S01_B、S02_B、S03_B、S04_B、S05_B
参考对应 A 版的排版，在 B 版上叠原生文本框
再铺大流程补图 B 版：P01_B、P02_B……
参考对应 A 版的排版，叠文字
套用你公司的 PPT 母版
微调字体、字号、位置，增删页

四、一句话理清对话策略

对话1 只读 → 读参考方案、读项目数据 → 输出文本

开新对话

对话2 只画 → 画小流程5张核心图 → 输出 AB 双版 PNG

开新对话

对话3 先规划再补画 → 先做映射 + 生成分页内容 → 同一个对话里接着补画大流程缺的页面 → 输出方案文本 + AB 双版 PNG

对话3 不能分裂——因为05依赖于04的映射结果，一旦开新对话，AI 就忘了哪些页面需要补图、哪些已经由小流程覆盖了。
